The Problem

Students are not allowed to select a project that has already been done in a previous CS410 semester. So they are required to look through pages and pages of reports from previous semesters. While they will get to see some examples of previous projects, this is ultimately a waste of time because most students will simply click on one project, read the first page and move on because its not related to the project they have in mind.

In some situations, students either don't look at all the projects, or suggest them anyway. In these situations, the professor is presented with a project that seems familliar, but there is still the question of when the project was done. So in this situation the professor much return to the list of previous projects and look through them all to find when it was done. This is time the professor could have spent teaching or reviewing other projects. Multiply that by the number of students suggesting repeat projects, and multiply again by how many semesters this has occured, and there is a lot of wasted time.


The Solution

Compile details on all previous projects and create a searchable archive that allows students to look up details on their project solution and the archive will return links to previous projects that might be related. Results should be categorized to aid in lookups. The system will have to "learn" from new projects that are created in the future so that they will be automatically be added to the database of project characteristics.


Competition

The current scope of this project is limited to the ODU Computer Science previous projects page, and therefore has a low risk of competition. Research should be done in order to find search providers that might be able to adapt their search engine in order to search the ODU archives.
